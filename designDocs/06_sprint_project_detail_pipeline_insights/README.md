\# Sprint 06 — Détail du Projet + Pipeline + Insights



\## 🎯 Objectif

Créer ou améliorer une page de détail d’un projet afin de représenter le cœur métier de la plateforme.



\## 📌 Contexte

Le frontend est déjà fonctionnel avec :

\- Authentification (login/register)

\- Dashboard

\- Projects

\- Analytics

\- Settings

\- Admin panel



La prochaine étape est de mettre en avant la logique métier principale à travers une page centrée sur le projet.



\---



\## 🧩 Fonctionnalités à ajouter



\### 1. Vue générale du projet

Afficher :

\- nom du projet

\- description

\- catégorie

\- statut

\- date de création



\---



\### 2. Concurrents

Afficher :

\- liste des concurrents

\- site web ou plateforme

\- statut de suivi



\---



\### 3. Pipeline (processus)

Afficher les étapes :

\- scraping

\- nettoyage

\- classification

\- analyse

\- insights



Avec pour chaque étape :

\- terminé (done)

\- en cours (running)

\- en attente (pending)

\- erreur (failed)



\---



\### 4. Insights (résultats)

Afficher :

\- opportunités principales

\- signaux importants des concurrents

\- tendances d’engagement

\- recommandations marketing



\---



\### 5. États UX

Gérer :

\- chargement (loading)

\- absence de données (empty state)

\- erreurs (error state)



\---



\## ⚙️ Contraintes

\- Ne pas modifier les autres pages

\- Respecter le design existant

\- Utiliser TypeScript propre

\- Garder une structure modulaire

\- Utiliser le backend si disponible, sinon mock propre



\---



\## 📦 Résultat attendu

Une page de détail projet claire et structurée permettant de visualiser :

\- le suivi d’un projet

\- l’avancement du pipeline

\- les résultats d’analyse



\---

Amélioration UI globale (UX, animations, design)
